MongoDB Basics Tutorial

Practice insert, find, update, and delete commands using the mongoadmin.people collection.

Before You Start

This tutorial assumes you already have a MongoDB database named mongoadmin and a collection named people.

Open mongosh and switch to the database:

use mongoadmin

Check the current database:

db

Check available collections:

show collections
If people does not exist yet, MongoDB will create it automatically when you insert the first document.

1. Insert Documents

Step 1: Insert one person

insertOne() adds a single document to the collection.

db.people.insertOne({
  name: "Asha",
  age: 24,
  city: "Mumbai",
  skills: ["JavaScript", "MongoDB"],
  active: true
})

Step 2: Insert many people

insertMany() adds multiple documents at once.

db.people.insertMany([
  {
    name: "Rahul",
    age: 29,
    city: "Delhi",
    skills: ["Python", "MongoDB"],
    active: true
  },
  {
    name: "Meera",
    age: 31,
    city: "Bengaluru",
    skills: ["Java", "Spring"],
    active: false
  },
  {
    name: "John",
    age: 27,
    city: "Pune",
    skills: ["Node.js", "React"],
    active: true
  }
])

Step 3: Insert a person with a nested address object

Documents can contain objects inside objects. This is useful when related details belong together.

db.people.insertOne({
  name: "Sneha",
  age: 26,
  email: "sneha@example.com",
  address: {
    street: "MG Road",
    city: "Bengaluru",
    state: "Karnataka",
    pincode: 560001
  },
  active: true
})

Step 4: Insert a person with an array of objects

An array can store simple values, or it can store multiple objects.

db.people.insertOne({
  name: "Vikram",
  age: 35,
  city: "Chennai",
  education: [
    {
      degree: "B.Tech",
      subject: "Computer Science",
      year: 2011
    },
    {
      degree: "MBA",
      subject: "Operations",
      year: 2015
    }
  ],
  active: true
})

Step 5: Insert a person with work experience details

This document uses nested objects, arrays, numbers, and booleans together.

db.people.insertOne({
  name: "Fatima",
  age: 32,
  city: "Hyderabad",
  job: {
    title: "Database Administrator",
    company: "CloudCore",
    yearsOfExperience: 7
  },
  certifications: ["MongoDB Associate", "AWS Cloud Practitioner"],
  remoteWorker: false,
  active: true
})

Step 6: Insert a person with a date field

Use new Date() when you want MongoDB to store a real date value.

db.people.insertOne({
  name: "Arjun",
  age: 28,
  city: "Jaipur",
  joinedAt: new Date("2025-01-15"),
  lastLoginAt: new Date(),
  active: true
})

Step 7: Insert a document with optional fields missing

MongoDB documents in the same collection do not need to have exactly the same fields.

db.people.insertOne({
  name: "Neha",
  city: "Kolkata",
  skills: ["HTML", "CSS"]
})

Step 8: Insert a student-style document

The collection is named people, but each person can have a different structure depending on what you need to store.

db.people.insertOne({
  name: "Kabir",
  type: "student",
  age: 21,
  city: "Ahmedabad",
  marks: {
    maths: 88,
    science: 92,
    english: 81
  },
  hobbies: ["Cricket", "Reading", "Coding"],
  active: true
})

Step 9: Insert many people with different structures

insertMany() can also insert documents that do not all share the same shape.

db.people.insertMany([
  {
    name: "Priya",
    type: "freelancer",
    city: "Goa",
    projectsCompleted: 18,
    skills: ["Design", "Figma"]
  },
  {
    name: "Sanjay",
    type: "manager",
    city: "Noida",
    team: {
      name: "Backend API",
      size: 6
    },
    active: true
  },
  {
    name: "Lina",
    type: "intern",
    college: "National Engineering College",
    stipend: 15000,
    skills: ["Python"]
  }
])

Step 10: Confirm inserted data

db.people.find()

For cleaner output, use pretty():

db.people.find().pretty()

2. Find Documents

Step 1: Find all people

db.people.find()

Step 2: Find one person

findOne() returns the first matching document.

db.people.findOne({ name: "Asha" })

Step 3: Find people from a city

db.people.find({ city: "Delhi" })

Step 4: Find active people

db.people.find({ active: true })

Step 5: Find people older than 25

$gt means greater than.

db.people.find({ age: { $gt: 25 } })

Step 6: Find people aged 25 to 30

$gte means greater than or equal to. $lte means less than or equal to.

db.people.find({
  age: { $gte: 25, $lte: 30 }
})

Step 7: Find people who know MongoDB

db.people.find({ skills: "MongoDB" })

Step 8: Show only selected fields

The second argument is called projection. Use 1 to include a field and 0 to hide _id.

db.people.find(
  { active: true },
  { _id: 0, name: 1, city: 1 }
)

Step 9: Sort results by age

Use 1 for ascending order and -1 for descending order.

db.people.find().sort({ age: 1 })

Step 10: Limit results

db.people.find().limit(2)

3. Update Documents

Step 1: Update one field

$set changes or adds fields without replacing the whole document.

db.people.updateOne(
  { name: "Asha" },
  { $set: { city: "Hyderabad" } }
)

Step 2: Update many documents

This marks all people from Pune as inactive.

db.people.updateMany(
  { city: "Pune" },
  { $set: { active: false } }
)

Step 3: Increase a number

$inc increases or decreases numeric values.

db.people.updateOne(
  { name: "Rahul" },
  { $inc: { age: 1 } }
)

Step 4: Add a skill to an array

$push adds a value to an array.

db.people.updateOne(
  { name: "Meera" },
  { $push: { skills: "Docker" } }
)

Step 5: Add a skill only if it does not already exist

$addToSet avoids duplicate array values.

db.people.updateOne(
  { name: "Asha" },
  { $addToSet: { skills: "MongoDB" } }
)

Step 6: Remove a field

$unset removes a field from matching documents.

db.people.updateOne(
  { name: "John" },
  { $unset: { active: "" } }
)

Step 7: Check the updated document

db.people.findOne({ name: "Asha" })
Be careful with updateMany(). If your filter is empty, it can update every document in the collection.

4. Delete Documents

Step 1: Delete one person

deleteOne() removes the first matching document.

db.people.deleteOne({ name: "John" })

Step 2: Delete many people

This removes all inactive people.

db.people.deleteMany({ active: false })

Step 3: Confirm what remains

db.people.find().pretty()
Avoid running db.people.deleteMany({}) unless you really want to delete every document in people.

5. Useful Practice Commands

Count all people

db.people.countDocuments()

Count active people

db.people.countDocuments({ active: true })

Find people whose city is either Mumbai or Delhi

$in matches any value from a list.

db.people.find({
  city: { $in: ["Mumbai", "Delhi"] }
})

Find people who are not active

$ne means not equal.

db.people.find({ active: { $ne: true } })

Find people with both JavaScript and MongoDB skills

$all checks that an array contains all listed values.

db.people.find({
  skills: { $all: ["JavaScript", "MongoDB"] }
})

7. Create a Large Practice Database

The project now includes a script named seed-bigone.js. It creates a database named bigone with five collections:

  • customers with 5,000 documents
  • products with 5,000 documents
  • orders with 5,000 documents
  • reviews with 5,000 documents
  • supportTickets with 5,000 documents
Running this script recreates the bigone database. If a bigone database already exists, its data will be deleted first.

Step 1: Open a terminal in this project folder

cd D:\new-projects\mongodb-course\single-node-cluster-installation

Step 2: Run the seed script

Make sure MongoDB is running, then run:

mongosh seed-bigone.js

Step 3: Switch to the new database

mongosh
use bigone

Step 4: Check the collections

show collections

Step 5: Count documents in each collection

db.customers.countDocuments()
db.products.countDocuments()
db.orders.countDocuments()
db.reviews.countDocuments()
db.supportTickets.countDocuments()

Step 6: Try a few sample queries

db.customers.findOne()
db.products.find({ category: "Laptop" }).limit(5)
db.orders.find({ status: "delivered" }).limit(5)
db.reviews.find({ rating: 5 }).limit(5)
db.supportTickets.find({ status: "open" }).limit(5)

8. Back Up the bigone Database Using mongoexport

mongoexport exports one collection at a time. To back up all five collections from bigone, run one command per collection.

Step 1: Create a backup folder

In PowerShell, run:

mkdir bigone-export

Step 2: Export customers

mongoexport --db bigone --collection customers --out bigone-export\customers.json --jsonArray

Step 3: Export products

mongoexport --db bigone --collection products --out bigone-export\products.json --jsonArray

Step 4: Export orders

mongoexport --db bigone --collection orders --out bigone-export\orders.json --jsonArray

Step 5: Export reviews

mongoexport --db bigone --collection reviews --out bigone-export\reviews.json --jsonArray

Step 6: Export support tickets

mongoexport --db bigone --collection supportTickets --out bigone-export\supportTickets.json --jsonArray

Step 7: Check exported files

dir bigone-export
If your MongoDB server uses authentication, add your connection string with --uri. Example: mongoexport --uri "mongodb://username:password@localhost:27017/bigone" --collection customers --out bigone-export\customers.json --jsonArray
mongoexport is good for JSON or CSV data exports. For a full database backup including indexes and metadata, use mongodump. For this tutorial, the commands above focus on mongoexport as requested.
